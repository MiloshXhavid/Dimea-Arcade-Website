<task type="checkpoint:human-action">
  <name>Task 1: Connect dimea-website repo to Vercel</name>

  <what-to-do>
  This task is done entirely in the Vercel dashboard — no CLI needed.

  **Before starting:** Confirm 05-01 is complete — `git branch -r` shows `origin/main` only, and `vercel.json` is committed.

  ---

  ### Step 1: Create a Vercel account (if not already done)

  Go to https://vercel.com and sign up. Use "Continue with GitHub" — this is required for the GitHub integration to work.

  ---

  ### Step 2: Import the dimea-website repository

  1. Go to https://vercel.com/new
  2. Click **"Add New Project"** → **"Import Git Repository"**
  3. If prompted, authorize the Vercel GitHub App. Grant access to **only** the `dimea-website` repository (no need for all-repo access).
  4. Select `dimea-website` from the list.

  ---

  ### Step 3: Configure the project settings on the import screen

  On the "Configure Project" screen, set the following exactly:

  | Setting | Value | Why |
  |---------|-------|-----|
  | Framework Preset | **Other** | No build step — flat HTML |
  | Root Directory | `.` (leave as default) | HTML files are in repo root |
  | Build Command | *(leave empty)* | No build |
  | Output Directory | `.` | Files served from repo root |
  | Install Command | *(leave empty)* | No package manager |

  > Do NOT select Next.js, Vite, or any other framework — they add build steps that will fail.

  ---

  ### Step 4: Deploy

  Click **"Deploy"**.

  Vercel will:
  - Clone the repo
  - Detect no build step
  - Deploy the HTML files from the root
  - Assign a `*.vercel.app` URL (e.g. `dimea-website-abc123.vercel.app`)

  The deployment should complete in under 60 seconds.

  ---

  ### Step 5: Verify the initial deploy

  Once deployment shows "Congratulations" / green status:

  1. Click the preview link Vercel shows (the `*.vercel.app` URL)
  2. Confirm the Dimea Arcade site loads correctly
  3. Test a clean URL: visit `[your-vercel-url]/about` — should load the site (not a 404)

  If `/about` returns 404, the `vercel.json` rewrites are not being applied. Check that `vercel.json` is committed on `main` before proceeding.
  </what-to-do>

  <resume-signal>
  Type "vercel connected" once the Vercel project is created and the initial deploy is green.
  Include the `*.vercel.app` URL Vercel assigned (e.g. `dimea-website-abc123.vercel.app`).
  </resume-signal>
</task>

<task type="checkpoint:human-action">
  <name>Task 2: Add custom domain dimea.io in Vercel</name>

  <what-to-do>
  Add both the apex domain and the www subdomain to the Vercel project, then get the exact DNS record values from the dashboard.

  ---

  ### Step 1: Open Vercel domain settings

  In your Vercel project → **Settings** → **Domains**

  ---

  ### Step 2: Add the apex domain

  1. Click **Add Domain**
  2. Type: `dimea.io`
  3. Click **Add**

  Vercel will ask how to handle `www.dimea.io`. Select **"Add www.dimea.io and redirect to dimea.io"** (or equivalent wording). This adds both domains at once and configures the www → apex redirect automatically.

  If Vercel does not offer this prompt, add `www.dimea.io` separately in Step 3.

  ---

  ### Step 3: Add www subdomain (if not already added in Step 2)

  1. Click **Add Domain** again
  2. Type: `www.dimea.io`
  3. Click **Add**
  4. On the `www.dimea.io` row, click **Edit**
  5. In the **"Redirect to"** dropdown, select `dimea.io`
  6. Click **Save**

  This configures a 308 permanent redirect at the Vercel CDN level — no `vercel.json` entry needed for this.

  ---

  ### Step 4: Read the exact DNS record values from Vercel

  After adding the domains, Vercel shows the required DNS records. Read these carefully — write them down.

  You will see something like:

  | Type | Name | Value |
  |------|------|-------|
  | A | @ | 76.76.21.21 |
  | CNAME | www | cname.vercel-dns-0.com (or a project-specific value) |

  > **IMPORTANT:** The CNAME value for www may be project-specific (e.g. `d1d4fc829fe7bc7c.vercel-dns-017.com`). Do NOT use a guessed value — read it directly from the Vercel dashboard here.

  You can also run this in your terminal to confirm:
  ```bash
  npx vercel domains inspect dimea.io
  ```
  (This requires being logged in with `npx vercel login` first.)
  </what-to-do>

  <resume-signal>
  Type "domains added" and paste the two DNS record values Vercel showed you:
  - A record value (should be `76.76.21.21`)
  - CNAME value for www (exact string from dashboard)
  </resume-signal>
</task>

<task type="checkpoint:human-action">
  <name>Task 3: Add DNS records at your domain registrar</name>

  <what-to-do>
  Log in to your domain registrar (wherever you registered `dimea.io`) and add the two DNS records.

  ---

  ### Records to add

  Use the exact values from Vercel's dashboard (confirmed in Task 2).

  **Record 1 — Apex domain (dimea.io)**

  | Field | Value |
  |-------|-------|
  | Type | A |
  | Name / Host | `@` (some registrars use blank or the domain name itself) |
  | Value / Points to | `76.76.21.21` |
  | TTL | 3600 (or "Automatic" / default) |

  **Record 2 — www subdomain**

  | Field | Value |
  |-------|-------|
  | Type | CNAME |
  | Name / Host | `www` |
  | Value / Points to | [exact CNAME value from Vercel dashboard] |
  | TTL | 3600 (or "Automatic" / default) |

  ---

  ### Registrar-specific notes

  - **The `@` symbol** means "the root/apex domain". Some registrars label it "blank", leave the Name field empty, or show the full domain name (`dimea.io`). All mean the same thing.
  - **If your registrar has existing A records** for `@` pointing elsewhere, remove or replace them with `76.76.21.21`.
  - **Do not add both an A record and a CNAME for `@`** — only the A record goes on `@`; CNAME goes on `www`.

  ---

  ### After adding records

  DNS propagation is usually complete within 5–30 minutes, though it can take up to 48 hours. Vercel polls automatically and updates the domain status to "Valid Configuration" once the records are detected.

  You can monitor propagation with:
  ```bash
  # Check A record for apex
  nslookup dimea.io
  # Expected: Address: 76.76.21.21

  # Check CNAME for www
  nslookup www.dimea.io
  # Expected: Alias to the Vercel CNAME value
  ```

  Or use https://dnschecker.org to see propagation across multiple DNS servers.

  Keep the Vercel dashboard → Settings → Domains tab open — it will show green checkmarks as each record propagates.
  </what-to-do>

  <resume-signal>
  Type "dns set" once both records are added to the registrar.
  You do not need to wait for full propagation before reporting — just confirm the records are saved.
  </resume-signal>
</task>

  | Command | Expected |
  |---------|----------|
  | `curl -I https://dimea.io` | `HTTP/2 200` |
  | `curl -I https://www.dimea.io` | `HTTP/1.1 308` with `Location: https://dimea.io` |
  | `curl -I https://dimea.io/about` | `HTTP/2 200` |
  | `curl -I https://dimea.io/hardware` | `HTTP/2 200` |
  | `curl -I https://dimea.io/impressions` | `HTTP/2 200` |
  | `curl -I https://dimea.io/shop` | `HTTP/2 200` |
  | `curl -I https://dimea.io/app` | `HTTP/2 200` |
  | `curl -I https://dimea.io/privacy.html` | `HTTP/2 200` |

  1. Visit https://dimea.io — site loads, no certificate warning
  2. Visit https://www.dimea.io — browser redirects to https://dimea.io (check address bar)
  3. Visit https://dimea.io/about — site loads (not a 404)
  4. Click the lock icon in your browser — certificate is valid, issued for `dimea.io`

  3. `git push origin test-preview`
  4. In Vercel dashboard → Deployments — you should see a new preview deployment appear within ~30 seconds
  5. Click it — unique `*.vercel.app` URL works
  6. Delete the test branch: `git push origin --delete test-preview && git checkout main && git branch -d test-preview`
  </how-to-verify>
