<task type="auto">
  <name>Task 3: Create vercel.json and push to main</name>
  <files>vercel.json</files>
  <action>
Create `vercel.json` in the repo root with the following exact content.

Rationale for explicit rewrites instead of wildcard:
- A wildcard `/(.*) → /index.html` would intercept ALL unmatched paths including future routes and Vercel internal paths
- `cleanUrls: true` strips `.html` from existing files — it does NOT create routes for non-existent paths like `/about`
- Explicit rewrites are precise: each clean URL maps to `index.html`; any unlisted path gets Vercel's default 404

Rationale for NOT including `/privacy`, `/terms`, `/refund` in rewrites:
- Those `.html` files exist on disk (`privacy.html`, `terms.html`, `refund.html`)
- Vercel serves real files first, before evaluating rewrites
- Users should access them as `/privacy.html` — the legal pages use `.html` extensions in all footer links

File content:

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "rewrites": [
    { "source": "/about",       "destination": "/index.html" },
    { "source": "/hardware",    "destination": "/index.html" },
    { "source": "/impressions", "destination": "/index.html" },
    { "source": "/shop",        "destination": "/index.html" },
    { "source": "/app",         "destination": "/index.html" }
  ]
}
```

After creating the file, commit and push:

```bash
git add vercel.json
git commit -m "feat(05-01): add vercel.json — clean URL rewrites for SPA sections"
git push origin main
```
