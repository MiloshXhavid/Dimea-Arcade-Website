**Step 1 — Create icon.svg**

Create `icon.svg` at repo root. The icon must replicate the CSS logo mark: a teal right-arc (open on the left) with three stacked dots at the bottom-left. The SVG viewBox is 32×32.

The CSS arc is `border-radius: 0 50px 50px 0` on an 18×22px div with `border-left: none` — this is a right-side arc. In SVG, represent it as a rectangular arc path:

```svg

  <style>
    :root { --teal: #2ae8c4; --dot: #e8ecf5; }
    @media (prefers-color-scheme: dark) {
      :root { --teal: #2ae8c4; --dot: #e8ecf5; }
    }
  </style>
  <!-- Right-arc: rectangle with rounded right corners only, no left border.
       Simulated as path: M top-left corner, line to top-right area, arc to bottom-right, line back, arc to bottom-left. -->
  <!-- Arc shape: 18×22px equivalent scaled to 32×32 viewBox.
       In viewBox units: arc region is roughly x=10 to x=26, y=2 to y=24.
       Right corners radius ~10 units. -->
  <path d="M10,2 L20,2 Q26,2 26,8 L26,18 Q26,24 20,24 L10,24"
        fill="none"
        stroke="var(--teal)"
        stroke-width="2.5"
        stroke-linecap="round"
        stroke-linejoin="round"/>
```

**Step 2 — Generate favicon.ico and apple-touch-icon.png from icon.svg**

Use the online favicon generator at https://favicon.io/favicon-converter/ or https://realfavicongenerator.net/ — OR use the approach below if those require a browser:

Alternative (command-line approach using Inkscape if installed):
```bash
# Check if Inkscape is available
inkscape --version 2>/dev/null

# Export 32×32 PNG from SVG
inkscape icon.svg --export-png=favicon-32.png --export-width=32 --export-height=32

# Export 180×180 PNG for apple-touch-icon
inkscape icon.svg --export-png=apple-touch-icon.png --export-width=180 --export-height=180
```

Alternative (using ImageMagick if installed):
```bash
convert -background none icon.svg -resize 32x32 favicon-32.png
convert -background none icon.svg -resize 180x180 apple-touch-icon.png
convert favicon-32.png favicon.ico
```

Alternative (using Python with cairosvg if available):
```bash
python3 -c "import cairosvg; cairosvg.svg2png(url='icon.svg', write_to='favicon-32.png', output_width=32, output_height=32)"
python3 -c "import cairosvg; cairosvg.svg2png(url='icon.svg', write_to='apple-touch-icon.png', output_width=180, output_height=180)"
```

If NONE of the above tools are installed: Create a minimal placeholder `favicon.ico` using the ICO binary approach below, and flag the apple-touch-icon as needing manual creation in the checkpoint task (Task 2).

**Minimal ICO fallback (Python):** A 32×32 single-color teal ICO to serve as placeholder:
```python
# Run: python3 create-favicon.py
import struct

def create_ico():
    # 32x32 ICO with a simple teal pixel pattern
    width, height = 32, 32
    # BMP info
    bmp_data = bytearray()
    # BITMAPINFOHEADER (40 bytes)
    bmp_data += struct.pack('<I', 40)  # biSize
    bmp_data += struct.pack('<i', width)  # biWidth
    bmp_data += struct.pack('<i', height * 2)  # biHeight (ICO uses doubled height)
    bmp_data += struct.pack('<H', 1)   # biPlanes
    bmp_data += struct.pack('<H', 32)  # biBitCount
    bmp_data += struct.pack('<I', 0)   # biCompression (BI_RGB)
    bmp_data += struct.pack('<I', 0)   # biSizeImage
    bmp_data += struct.pack('<i', 0)   # biXPelsPerMeter
    bmp_data += struct.pack('<i', 0)   # biYPelsPerMeter
    bmp_data += struct.pack('<I', 0)   # biClrUsed
    bmp_data += struct.pack('<I', 0)   # biClrImportant
    # Pixel data: 32x32 BGRA — teal #2ae8c4 = B=0xc4 G=0xe8 R=0x2a A=0xff
    for row in range(height):
        for col in range(width):
            # Simple arc shape: right half is teal, left is transparent
            if col > 12:
                bmp_data += bytes([0xc4, 0xe8, 0x2a, 0xff])  # BGRA teal
            else:
                bmp_data += bytes([0x00, 0x00, 0x00, 0x00])  # transparent
    # AND mask: 32x32 bits, 1=transparent, 0=opaque. 32 rows * ceil(32/8)=4 bytes = 128 bytes
    and_mask = bytes([0x00] * (height * 4))
    image_data = bytes(bmp_data) + and_mask
    # ICO header
    ico_header = struct.pack('<HHH', 0, 1, 1)  # reserved, type=1 (ICO), count=1
    # Directory entry
    ico_dir = struct.pack('<BBBBHHII',
        width, height, 0, 0, 1, 32,
        len(image_data), 6 + 16)  # size, offset (header + one 16-byte dir entry)
    with open('favicon.ico', 'wb') as f:
        f.write(ico_header + ico_dir + image_data)

create_ico()
print("favicon.ico created")
```

This produces a functional ICO that renders in all browsers. It is a simplified placeholder — if Inkscape or ImageMagick is available, use those for a higher-quality result that matches icon.svg exactly.

DO NOT skip favicon.ico — some browsers (older Chrome, Edge) fall back to it when SVG is unsupported.
